ARES FTC Robot Code Documentation

Welcome to the ARES FTC robot code documentation. This codebase implements a command-based architecture for swerve drive with advanced sensor fusion and vision localization.

Architecture Overview

The robot code is organized into several key packages:

  • {@link org.firstinspires.ftc.teamcode.subsystems}: Subsystems including drive, encoders, and other robot components
  • {@link org.firstinspires.ftc.teamcode.commands}: Commands for robot control
  • {@link org.firstinspires.ftc.teamcode.util}: Utility classes for motor control, sensor fusion, data logging, and calibration
  • {@link org.firstinspires.ftc.teamcode.hardware}: Hardware drivers for custom devices like the SRS Hub
  • {@link org.firstinspires.ftc.teamcode.opmode}: FTC OpModes for autonomous and teleop control

Key Components

Swerve Drive System

The robot uses an octo-swerve drive configuration with four independently controlled modules. Each module can rotate its wheel direction (steering) and drive at variable speeds. Key classes:

  • {@link org.firstinspires.ftc.teamcode.subsystems.drive.OctoSwerveDrivetrainV2}: Main drivetrain controller
  • {@link org.firstinspires.ftc.teamcode.subsystems.drive.OctoSwerveModuleV2}: Individual swerve module with feedforward and PID control
  • {@link org.firstinspires.ftc.teamcode.subsystems.encoders.SwerveEncoder}: Swerve encoder interface with multiple implementations

Motor Control

Advanced motor control techniques ensure smooth, efficient operation:

  • {@link org.firstinspires.ftc.teamcode.util.control.SlewRateLimiter}: Prevents sudden power changes that cause current spikes
  • {@link org.firstinspires.ftc.teamcode.util.control.CurrentLimiter}: Monitors current draw and scales motor power to prevent tripping the 20A fuse

Sensor Fusion

Multiple sensors are combined using an Extended Kalman Filter for accurate localization:

  • Swerve Odometry: Position estimation from wheel encoders
  • GoBilda Pinpoint: Deadwheel odometry with IMU sensor fusion
  • Limelight Vision: AprilTag-based vision localization with multi-camera fusion
  • {@link org.firstinspires.ftc.teamcode.util.localization.SensorFusionLocalizer}: Combines all sensors using EKF

Configuration

All robot-specific constants are centralized in:

  • {@link org.firstinspires.ftc.teamcode.Constants}: Physical dimensions, PIDF coefficients, feedforward constants, and tuning parameters
  • {@link org.firstinspires.ftc.teamcode.Robot}: Robot singleton with hardware initialization and subsystem access

Getting Started

For New Developers

  1. Review the {@link org.firstinspires.ftc.teamcode.Constants} class to understand robot configuration
  2. Study {@link org.firstinspires.ftc.teamcode.subsystems.drive.Drive} to see how the drive subsystem integrates everything
  3. Check {@link org.firstinspires.ftc.teamcode.opmode.BaseTeleOp} for the standard TeleOp OpMode structure

For Tuning and Calibration

The codebase includes comprehensive tuning and calibration tools:

  • {@link org.firstinspires.ftc.teamcode.opmode.autonomous.AutoTuneFeedforward}: Automatically tune kS and kV feedforward constants
  • {@link org.firstinspires.ftc.teamcode.opmode.autonomous.AutoTuneSteering}: Automatically tune steering PID
  • {@link org.firstinspires.ftc.teamcode.util.tuning.SteeringTuner}: Manual steering PID tuning with real-time feedback
  • {@link org.firstinspires.ftc.teamcode.util.tuning.FeedforwardTuner}: Manual feedforward tuning with velocity testing

Control Theory

Feedforward Control

The swerve drive uses feedforward control to compensate for friction and maintain target speeds: 

motorPower = kS × sign(velocity) + kV × velocity
  • kS (Static Friction): Minimum power to overcome friction
  • kV (Velocity Constant): Power to maintain velocity

PID Control

PID feedback corrects errors between target and actual states: 

correction = Kp × error + Ki × ∫error dt + Kd × d(error)/dt

Current Limiting

To prevent tripping the 20A fuse, the drivetrain uses:

  • Slew Rate Limiting: Prevents rapid power changes
  • Current Monitoring: Scales down power when approaching limits
  • Soft Recovery: Gradually restores power when current drops

Telemetry and Logging

Comprehensive telemetry and data logging are available:

  • {@link org.firstinspires.ftc.teamcode.util.logging.DataLogger}: CSV logging to disk for post-match analysis
  • Panels Dashboard integration for real-time visualization
  • Per-module motor power breakdown (feedforward, PID, total)
  • Battery voltage monitoring
  • Loop timing and performance metrics

Hardware Requirements

  • Control Hub: FTC Robot Controller (Rev Control Hub or similar)
  • Drive Motors: 4x brushless motors with encoders
  • Steering Servos: 4x high-torque servos (e.g., Melon Super Servo)
  • Encoders: OctoQuad or analog absolute encoders for steering
  • IMU: Built-in Control Hub IMU or external IMU
  • Odometry: GoBilda Pinpoint or SRS Hub with deadwheels
  • Vision: Limelight3A camera(s) for AprilTag localization
